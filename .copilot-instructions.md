# ✅ LOGGING AUTOMÁTICO CONFIGURADO

## 🎯 Sistema de Hook

Todos os prompts são registrados **automaticamente** via hook no evento `userPromptSubmitted`:
- ✅ Sem necessidade de chamar `/log-copilot-prompts` manualmente
- ✅ Cada prompt é capturado e salvo em `copilot-prompts.json`
- ✅ Funciona transparentemente para todas as interações

### Como Funciona
1. Usuário submete um prompt
2. Hook `userPromptSubmitted` é acionado automaticamente
3. Script `.github/hooks/scripts/log-prompt-hook.sh` executa
4. Prompt é registrado com timestamp, branch Git e contexto
5. Resposta continua normalmente

### Configuração
- **Hook File**: `.github/hooks/hooks.json`
- **Script**: `.github/hooks/scripts/log-prompt-hook.sh`
- **Evento**: `userPromptSubmitted` (cada prompt submetido)
- **Log Output**: `copilot-prompts.json` (raiz)

## 📝 Orientações de Contexto

O contexto do prompt é registrado automaticamente para categorização. Use uma descrição breve que identifique:
- **Camada**: database, api, messaging, etc.
- **Componente**: controllers, routes, services, models, etc.
- **Tipo de trabalho**: implementation, refactoring, debugging, optimization, etc.
- **Tema**: authentication, validation, error handling, architecture, etc.

## � Dados Registrados Automaticamente

Cada prompt é salvo em `copilot-prompts.json` com:
- `id`: identificador único
- `timestamp`: ISO 8601 UTC
- `gitBranch`: branch Git atual
- `gitUser`: usuário Git configurado
- `promptText`: texto completo do prompt
- `context`: contexto detectado
